# Plan: Multiple Reference Files for Coaching Scenarios

## Goal
Allow super admin to upload multiple reference files when creating/editing a coaching scenario in the super admin panel.

## Current State
- Single file upload supported via `reference_file_name` and `reference_file_text` columns in `coaching_scenarios` table
- Text directly injected into system prompt (truncated to 12K chars)

## Required Changes

### 1. Database Model (`agente_rolplay/db/models.py`)

Add new table `CoachingScenarioReferenceFile`:

```python
class CoachingScenarioReferenceFile(Base):
    __tablename__ = "coaching_scenario_reference_files"

    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    scenario_id = Column(
        UUID(as_uuid=True),
        ForeignKey("coaching_scenarios.id", ondelete="CASCADE"),
        nullable=False,
    )
    file_name = Column(String(255), nullable=False)
    file_text = Column(Text, nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow)

    scenario = relationship("CoachingScenario", back_populates="reference_files")
```

Add relationship to `CoachingScenario`:
```python
reference_files = relationship("CoachingScenarioReferenceFile", back_populates="scenario", cascade="all, delete-orphan")
```

### 2. Database Migration (`agente_rolplay/db/database.py`)

Add migration to create new table:
```sql
CREATE TABLE coaching_scenario_reference_files (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    scenario_id UUID REFERENCES coaching_scenarios(id) ON DELETE CASCADE,
    file_name VARCHAR(255) NOT NULL,
    file_text TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT now()
);
```

### 3. Admin Router Updates (`agente_rolplay/routers/admin.py`)

**Update `admin_create_scenario` (POST /scenarios)**:
- Accept multiple files in form data (replace single `reference_file`)
- Loop through uploaded files, call `_extract_reference_file()` for each
- Create `CoachingScenarioReferenceFile` records for each file
- Remove old columns: `reference_file_name`, `reference_file_text` from response

**Update `admin_update_scenario` (PATCH /scenarios/{scenario_id})**:
- Accept additional files to add
- Handle optional file deletion via `delete_reference_files` array in body
- Update file count in response

**Add new endpoints**:
- `DELETE /scenarios/{scenario_id}/reference-files/{file_id}` - Delete single reference file
- `GET /scenarios/{scenario_id}/reference-files` - List all reference files for a scenario

### 4. Message Processor Updates (`agente_rolplay/messaging/message_processor.py`)

**Update `_compose_coaching_prompt`**:
- Accept `scenario_id` instead of `reference_file_text`
- Query all `CoachingScenarioReferenceFile` records for the scenario
- Concatenate with clear separators:

```
=== Reference File: {file_name_1} ===
{file_text_1}

=== Reference File: {file_name_2} ===
{file_text_2}
...
```

- Consider chunking/injection strategy for large total text ( token limits)

### 5. Frontend Updates (if needed)

- Update super admin panel to support multi-file upload
- Show list of uploaded files with delete option

## Execution Order

1. Add model to `models.py`
2. Add migration to `database.py`
3. Update admin router - POST/PATCH endpoints
4. Update admin router - new DELETE/GET endpoints
5. Update message processor
6. Test end-to-end

## Notes
- Keep backwards compatibility: existing scenarios with single file in main table should still work
- Consider lazy migration script to move existing `reference_file_text` to new table
